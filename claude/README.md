# 営業日報システム

## プロジェクト概要

営業担当者が日々の営業活動を記録し、上長が確認・コメントできる日報管理システムです。

### 主な目的

- 営業活動の可視化と記録の一元管理
- 上長による効果的なフィードバックとコミュニケーション
- 日報作成・確認プロセスの効率化

## 主要機能

### 1. 日報管理機能

- **日報作成**: 営業担当者が日次で作成
- **訪問記録**: 1日に複数の顧客訪問を記録可能
  - 訪問した顧客
  - 訪問内容・結果
  - 訪問日時
- **Problem（課題・相談）**: 当日の課題や上長への相談事項を記載
- **Plan（明日の予定）**: 翌日の活動計画を記載

### 2. 承認・コメント機能

- 上長による日報の確認
- Problem、Planに対するコメント機能
- コメント確認機能
  - 各コメントに「確認済みにする」ボタンを配置
  - 確認済みコメントと未確認コメントを区別して表示
  - 未確認コメント数をホーム画面に表示

### 3. マスタ管理機能

- **顧客マスタ**: 顧客情報の管理
- **営業マスタ**: 営業担当者情報の管理

## ユーザー種別

- **営業担当者**: 日報の作成・編集、コメントの確認
- **上長（管理者）**: 日報の閲覧・コメント、マスタ管理

## クイックスタート

```bash
# リポジトリのクローン
git clone <repository-url>
cd sales_daily_report

# 依存パッケージのインストール
npm install

# 環境変数の設定
cp .env.example .env.local
# .env.local を編集して必要な環境変数を設定

# Prismaのセットアップ
npx prisma generate
npx prisma db push

# 開発サーバーの起動
npm run dev
```

詳細は [セットアップガイド](./setup.md) を参照してください。

## 技術スタック（概要）

- **フロントエンド**: Next.js (App Router), React, TypeScript
- **UIコンポーネント**: shadcn/ui + Tailwind CSS
- **バックエンド**: Next.js API Routes (Hono)
- **データベース**: MongoDB + Prisma
- **テスト**: Vitest
- **デプロイ**: Google Cloud Run

詳細は [技術スタック](./tech-stack.md) を参照してください。
